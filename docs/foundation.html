Foundation

Key fundamental capabilities of the UI library.

Architecture

The library follows a native-first architecture.

The library does not use Shadow DOM.

The library does not rely on CSS classes.

Styling

All components are customizable via CSS.

Components fully support both LTR and RTL directions.

ARIA

Most ARIA attributes are set automatically to reflect correct roles and component states.

In some cases, aria-label will need to be set manually.

Accessibility

Interactive elements support keyboard navigation: Tab to move forward, Shift + Tab to move backward, Enter or Space to trigger actions, and arrow keys where contextually relevant (e.g., in tabs).
