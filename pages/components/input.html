Input

Input allows users to enter and edit text.

Info

Static init-only attributes are removed from the custom element itself immediately after initialization. This prevents confusion and avoids conflicts between attributes and properties during the component lifecycle. Their values are still applied to the relevant internal elements, so the component's behavior remains consistent with the initial configuration.

To update these values dynamically, use the corresponding properties on the component instance.

The clickable area of the Clear button is larger than the icon itself to make it easier to click or tap.

Attributes

aria-label — static (required). If used outside of the field component, the aria-label must be set manually.

type — static (init-only, optional). Default: text. Specifies the input type. Update via property: component.type = 'email'.

value — static (init-only, optional). Sets the current value of the input. Update via property: component.value = 'Value'.

placeholder — static (init-only, optional). Text displayed when the input is empty. Update via property: component.placeholder = 'Placeholder'.

clearable — static (optional). Adds a clear button to the component.

required — reactive (optional). Indicates whether the component is required.

disabled — reactive (optional). Controls the disabled state of the component.

shape — static (optional). Default: rounded. Available options: rounded, pill, square. Sets the corner style.
