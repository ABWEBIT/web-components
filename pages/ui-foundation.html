Foundation

Key fundamental capabilities of the UI library.

Architecture

  • The library does not use Shadow DOM.
  • No internal CSS class names are required or exposed.
  • Text direction (LTR/RTL) inherits from the parent or system unless overridden.

Styling

  • All components are easily customizable via CSS.

Attributes

  • ARIA attributes reflect the correct roles and component states.

Accessibility

  • Interactive elements support keyboard navigation: Tab to move forward, Shift + Tab to move backward, Enter or Space to trigger actions, and arrow keys where contextually relevant (e.g., in tabs, menus, sliders).
